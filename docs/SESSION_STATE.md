## Что сделано за сессию

- Создана папка `~/croissant-model/`
- `index.html` — полностью адаптированная копия Sulvara Prospera v5 под импорт круассанов:
  - Палитра: espresso + caramel (замена значений в `:root`, имена CSS-переменных сохранены)
  - Брендинг: Croissant Import / Frozen Viennoiserie
  - Валюта: USD → EUR везде
  - Единица объёма: тонны → кг
  - `A_DEFAULT` под импорт: курс 580 €/тг, 200 т/год (рост до 560), пошлина 12.5%, НДС 12%, потери 3.5%
  - `cm()` переписана: закупка + фрахт + пошлина + брокер + РК-логи + хранение. Списания учтены в выручке (продаём только `vol × (1 − losses)`), но стоимость закупки — от полного объёма
  - `HR_DEFAULT` — 5 блоков, ~20 позиций команды импортёра
  - `risks` — 12 рисков под импорт
  - `RM_DEFAULT` — 17 задач, фазы Подготовка/Запуск/Таможня/Продажи/Рост
  - `renderCapex` — EU (рефрижераторы) + РК (склад+бренд+офис)
  - `renderInvest` — 3 фазы финансирования, сравнение с депозитом/облигациями/недвижимостью
- Git: init → commit → создан публичный репо `DaurenBol/croissant-import` → push

- Live: https://daurenbol.github.io/croissant-import/

## На чём остановились

Всё готово к показу. Следующий логичный шаг — **калибровка цифр под реальность** (см. `PLAN.md` → v1.1). Но это требует от пользователя внешних данных (ставка пошлины, цены у поставщиков, аренда склада, отпускные цены сетей).

## Открытые вопросы к пользователю

- Когда показ клиенту? От этого зависит, насколько срочно нужна калибровка цифр.
- Клиент уже работает с каким-то конкретным поставщиком в ЕС? (Франция / Бельгия / Польша / Турция?)
- Канал сбыта приоритетный — сети, HoReCa или своя розница? Сейчас стоит смесь 45/35/20 в структуре продаж.
- Масштаб первого года: 200 т — это реалистично или нужно меньше для MVP (50-100 т)?

## Потенциальные блокеры для следующей сессии

- Если клиент попросит **собственный бренд/лого** вместо «Croissant Import» — поменять `logo-badge`, `logo-name`, `logo-sub` (HTML ~строка 200) + `<title>` + возможно favicon.
- Если спросит **«почему у меня в сравнении NPV меньше чем в Excel»** — скорее всего расхождение в учёте НДС-ввозного (возмещается, но временно оттока кэш) или в потерях (списанный объём всё равно оплачен).
- Если поменяет структуру HR/дорожной карты через UI и захочет зафиксировать — тогда переносить значения в `HR_DEFAULT` / `RM_DEFAULT` руками, и бампать `v1` → `v2` в ключах localStorage.
