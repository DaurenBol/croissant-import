# DESIGN.md — палитра, шрифты, компоненты

## Стратегия

Тёмная премиум-тема в стилистике французской boulangerie. Задача — «серьёзный финтех-дашборд, но с тёплой круассановой душой». Избегаем: cartoonish, розовое, слишком светлое, слишком много эмодзи. Оставляем: контраст, монотипы для цифр, чёткие KPI-карты, сдержанные акценты.

## Палитра

Имена CSS-переменных **сохранены от Sulvara Prospera** (`--navy`, `--teal`, `--gold`, `--red`). Менялись только значения. Это позволило не переписывать 1000+ мест применения.

```css
:root {
  /* фоны */
  --navy:  #1A1308;   /* deep espresso       (было #0A1628 navy) */
  --navy2: #241A0E;   /* coffee bean         (было #111E35)       */
  --navy3: #2E2314;   /* warm brown          (было #162440)       */
  --navy4: #3B2D1C;   /* caramel shadow      (было #1D2F4F)       */

  /* акценты */
  --teal:  #D4A574;   /* caramel             (было #00C4E0 teal)  */
  --gold:  #E8C077;   /* butter gold         (было #F5C842)        */
  --green: #8FA857;   /* muted olive         (было #1DB87A)        */
  --red:   #C85A3E;   /* terracotta          (было #E84040)        */

  /* текст */
  --text:  #F4ECD8;   /* cream                                     */
  --text2: #BBA888;   /* warm beige                                */
  --text3: #7A6850;   /* darker tan                                */
  --muted: #6B5844;   /* tan                                       */

  /* границы */
  --border:  rgba(212,165,116,0.10);
  --border2: rgba(212,165,116,0.20);
}
```

### Правило при смене палитры

1. Меняешь значения в блоке `:root`.
2. В JS есть инлайн `rgba(...)` / `'#HEX'` значения для чартов — нужно заменить их через `Edit replace_all`:
   - `rgba(0,196,224,…)` (teal), `rgba(29,184,122,…)` (green), `rgba(245,200,66,…)` (gold), `rgba(232,64,64,…)` (red)
   - В этом проекте они уже переопределены в `rgba(212,165,116,…)`, `rgba(143,168,87,…)`, `rgba(232,192,119,…)`, `rgba(200,90,62,…)`. Если хочешь снова изменить — массовая замена по числовым rgba-значениям.

### Вспомогательные цвета (для фаз/категорий)

Используются в HR-блоках и Дорожной карте:

- `#A67A4B` (Подготовка) — тёмный caramel
- `#B89878` (Рост / Транспорт) — tan
- `#8B5A2B` (save-btn) — roast coffee

## Шрифты

- **Основной**: `IBM Plex Sans` (weights: 300/400/500/600, также italic 300)
- **Моно**: `IBM Plex Mono` (400/500) — для всех цифр, таймстампов, формул
- Грузятся с Google Fonts через `<link>` в head

Правило: **любые цифры → моно**, чтобы колонки в таблицах выравнивались. Это в CSS уже зашито через класс `.cv`, `.fval`, `.kc-val` и т.п.

## Компоненты

### KPI-карта (`.kc`)

```html
<div class="kc g">
  <div class="kc-label">Выручка</div>
  <div class="kc-val">1 234 млн</div>
  <div class="kc-sub">€2.1K</div>
</div>
```

Модификаторы: `.g` (зелёный акцент), `.a` (gold), `.r` (красный), `.w` (нейтральный). Цвет-акцент — в `::before` (верхняя полоска 2px) и в `.kc-val`.

### Чарт-бокс (`.cb`)

```html
<div class="cb">
  <div class="ct">Название чарта</div>
  <div class="leg">…легенда…</div>
  <div class="cw"><canvas id="..."></canvas></div>
</div>
```

`.cw` даёт фикс. высоту (195px), `.cw.tall` — 255px.

### Таблица-блок (`.tb`)

Тёмная таблица с sticky-like хедером. Моно-font для всех td кроме первого (label). Цвет `.neg` / `.pos` для значений.

### Слайдер-поле (`.field`)

Используется в «Допущениях». Range-input + live-значение в моно.

## Анимации

Минимальные. Из оригинала сохранены:

- `.kc:hover` — transform: translateY(−1px) + shadow
- `.panel` — fadeUp(.22s) при активации
- `.save-btn.saved` — savePulse (glow + scale) на 1.8 сек

Про «slow glass effects, visible images» (из preferences пользователя) — пока не применяли. Если захочет больше «воздуха» / блюр-эффектов — добавим `backdrop-filter: blur(...)` на `.cb` и `.kc`.

## UX-принципы

- **Save-button pattern**: слайдеры в «Допущениях» меняют `S` (staged), расчёт использует `A` (committed). Кнопка «Сохранить допущения» копирует `S → A` и вызывает `renderAll()`. Это защита от лаг-калькуляций при перетаскивании слайдера.
- **localStorage для UGC**: HR-штат и дорожная карта сохраняются на клиенте. Ключи `croissant_hr_v1` и `croissant_roadmap_v1`. **Бампать версию при смене defaults**.
- **Mobile-first**: @media (max-width: 768px) прячет sidebar в off-canvas, упрощает grids в 1 колонку.
- **Печать**: есть `@media print` — скрывает sidebar + topbar. Можно сделать PDF через Cmd+P.

## Иконки

SVG inline в сайдбаре — минималистичные, 15×15, currentColor. Каждый пункт меню имеет свою иконку. **Не ставь emoji в меню** — они ломают вертикальный ритм.

Emoji допустимы только в CAPEX-заголовках (🚚, 🥐) и Roadmap-фильтрах — там они функциональные.
