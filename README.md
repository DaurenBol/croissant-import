# Croissant Import — финмодель дашборда

Интерактивный финансовый дашборд для бизнес-идеи **«импорт замороженной виеннуазери из ЕС в Казахстан»** (круассаны, pain au chocolat, бриоши, кранчи и т.п.).

- **Live**: https://daurenbol.github.io/croissant-import/
- **Repo**: https://github.com/DaurenBol/croissant-import
- **Локально**: `~/croissant-model/`

## Что это

Одностраничный SPA без сборки — чистый HTML + inline CSS + vanilla JS + Chart.js через CDN. Открывается двойным кликом на `index.html`.

Архитектура и UX скопированы с дашборда **Sulvara Prospera** (завод H₂SO₄) и переделаны под бизнес-модель импорта круассанов. Клиенту понравилась именно эта компоновка — 12 связанных разделов с live-пересчётом всей модели от любого параметра в «Допущениях».

## 12 разделов

| Раздел | Что внутри |
|---|---|
| **Дашборд** | KPI (Объём, Выручка, EBITDA, NPV, IRR, BEP, Себестоимость), графики выручки и FCF, структура OPEX |
| **Допущения** | 20+ слайдеров: курс €/тг, цена закупки, фрахт, пошлина, НДС, логистика РК, хранение, потери, цена продажи, CAPEX. Кнопка **Сохранить** → пересчёт всей модели |
| **P&L (помесячный)** | 12 мес × 5 лет, ramp-кривая первого года, годовая сводка |
| **CAPEX** | EU-часть (рефрижераторы) + РК (склад + бренд + маркетинг + офис + ИТ), с доньют-диаграммой |
| **ДДС — Ввод** | Операц./Инвест./Финанс., авто или ручной ввод, live NPV |
| **ДДС — Таблица** | Полная ДДС по годам + накопленный FCF |
| **NPV / IRR** | Инвест-метрики + формулы |
| **Сценарии** | Стресс / Базовый / Оптимист по цене продажи, закупке €, курсу, фрахту |
| **Чувствительность** | Торнадо-диаграмма, BEP, прибыль по объёму |
| **HR & Команда** | ~20 чел: Управление, ВЭД/логистика, Продажи, Склад, Финансы. Add/edit/delete. Сохранение в localStorage |
| **Матрица рисков** | 12 рисков импортёра (P×I, heatmap, митигация) |
| **Инвестиционный обзор** | Полная потребность в финансировании, 3 фазы, калькулятор доли инвестора, таймлайн возврата, сравнение с депозитом/облигациями/недвижимостью |
| **Дорожная карта** | 17 задач от регистрации ТОО до выхода на сети. Фильтры, статусы, ответственные. Сохранение в localStorage |

## Быстрый старт

```sh
open -a "Google Chrome" ~/croissant-model/index.html
```

Или — любое изменение в модели правится в разделе **«Допущения»** через слайдеры → кнопка **«Сохранить допущения»** → вся модель пересчитывается.

Все defaults лежат в `index.html` в константе `A_DEFAULT` (~строка 355). Для «жёсткой» настройки (другой продукт, другая страна, другой объём) — правятся либо там, либо через UI.

## Деплой

Репо публичный на GitHub. Pages включены (`main` / `/`). Любой `git push origin main` обновляет live-версию через 30-90 секунд.

```sh
cd ~/croissant-model
git add <files>
git commit -m "..."
git push
```

## Кредиты

Архитектура дашборда — форк логики **Sulvara Prospera v5** (тот же каркас из 12 разделов, та же функция `cm()` как ядро). Вся бизнес-логика, параметры, HR-штат, риски и дорожная карта переписаны под импорт круассанов с нуля.
